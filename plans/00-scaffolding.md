# Plan 00 — Package scaffolding & shared infrastructure

## Objective

Stand up an installable, checkable, testable `meteoTidy` package skeleton and the
handful of cross-cutting helpers every later plan uses: the condition system,
the injectable-clock helper, and the lint/CI config. No domain logic yet.

## Scope

**In:**
- `DESCRIPTION`, `NAMESPACE` (roxygen-managed), package-level doc, `LICENSE`.
- testthat 3 harness and CI.
- Condition helpers (`abort_meteo()`, `warn_meteo()`, `inform_meteo()`).
- Clock helper (`.now()`), and a test helper to freeze it.
- `.lintr`, `NEWS.md`, `_pkgdown.yml` stub.

**Out (do NOT build here):**
- Any schema, adapter, storage, or statistics code. Those are Plans 01+.
- Config/secret loading (Plan 14).

### `R/meteoTidy-package.R`

### `R/conditions.R` — the condition system

Every user-facing error/warning/message in the package goes through these. They
wrap `cli` and attach a stable class so tests and callers can catch precisely.

```r
abort_meteo(message, ..., class, call = rlang::caller_env(), .envir = parent.frame())
warn_meteo(message, ..., class, .envir = parent.frame())
inform_meteo(message, ..., class = NULL, .envir = parent.frame())
```

Behaviour:
- `message` is a `cli`-formatted character vector (supports `{.val }`, bullet
  names `"i"`, `"x"`, `"!"`), passed straight to `cli::cli_abort()` /
  `cli::cli_warn()` / `cli::cli_inform()`.
- `class` is **required** for `abort_meteo` and `warn_meteo`. The helper prepends
  `"meteoTidy_error_"` / `"meteoTidy_warning_"` so callers pass the short kind
  (`class = "bad_units"` → full class `"meteoTidy_error_bad_units"`). Also attach
  the umbrella classes `"meteoTidy_error"` / `"meteoTidy_warning"` so a caller can
  catch all of them.
- Pass `call` through to `cli` so the reported call is the caller, not the helper.
- `inform_meteo`’s `class` is optional; when given, prefix the same way.

Provide one documented, exported helper `meteo_conditions()` returning a small
tibble/data.frame of `(class, meaning)` for the classes the package can raise,
so the taxonomy is discoverable and testable. Each later plan appends its
classes here (the plan text will say which).

### `R/utils.R`

- `.now(tz = "UTC")` — returns `Sys.time()` coerced to `tz`. **This is the only
  place in the package that reads the wall clock.** All later code that needs
  “now” calls `.now()` (or, better, takes `now = .now()` as an argument). This
  is the seam tests freeze.
- `%||%` re-export if not already from rlang.
- Keep this file tiny; it is not a dumping ground.

### Test helper: freezing the clock

`tests/testthat/helper-conditions.R` provides:
- `local_frozen_clock(t = as.POSIXct("2026-07-05 00:00:00", tz = "UTC"), env = parent.frame())`
  — uses `testthat::local_mocked_bindings(.now = function(tz = "UTC") ...)` so any
  code under test that calls `.now()` sees a fixed instant. Document that
  functions must call `.now()` (not `Sys.time()`) for this to work.

## Test requirements

### `test-conditions.R`
- `abort_meteo("boom", class = "bad_units")` raises an error whose class vector
  contains both `"meteoTidy_error_bad_units"` and `"meteoTidy_error"`. Assert via
  `expect_error(..., class = "meteoTidy_error_bad_units")`.
- The reported `call` is the calling function, not `abort_meteo` itself
  (construct a wrapper `f()` that calls `abort_meteo`, catch the condition, and
  assert `conditionCall` / the `rlang::cnd_call` is `f`).
- `warn_meteo` produces a warning with the parallel class; `expect_warning(...,
  class = "meteoTidy_warning_x")`.
- `abort_meteo` with no `class` is itself an error (fail fast on misuse).
- Snapshot one formatted multi-bullet message with `expect_snapshot()` to lock
  the cli rendering (inject any variable content as fixed values).
- `meteo_conditions()` returns a data frame with unique `class` values, each
  starting with `"meteoTidy_"`.

### `test-utils.R`
- `.now()` returns a `POSIXct` with `attr(, "tzone") == "UTC"`.
- With `local_frozen_clock()` active, `.now()` returns exactly the frozen instant
  — proving the seam works for downstream plans.

### Global test setup
- `tests/testthat/helper-conditions.R` also sets, via `setup`/`teardown` or a
  `helper`, an option or env var (e.g. `withr::local_envvar(METEOTIDY_NO_NET =
  "1")` in a `setup-*.R`) that Plan 04’s HTTP seam will honour to turn any live
  request into an error. Define the env var name here; document it in the README
  testing section (already referenced there).

## Definition of done

Shared skeleton (README) plus:
- `devtools::check()` passes clean on a fresh clone (0 errors/warnings/notes
  beyond the expected “new package” notes).
- `.now()` is the sole reader of the wall clock (grep for `Sys.time`/`Sys.Date`
  in `R/` returns only the body of `.now()`).
- The condition helpers are used by no domain code yet, but are exported,
  documented, and fully tested.
- `NEWS.md` started with an “initial scaffolding” bullet under a `# meteoTidy
  (development version)` heading.
